# Flujo conversacional

## 1. Diagnóstico de inicio

Identifica el material disponible y selecciona una ruta.

| Si el usuario trae… | Ruta inicial |
|---|---|
| Idea, emoción, título, frase o imagen | Brief → estilo → estructura → hook → letra |
| Letra parcial | Diagnóstico → completar estructura → revisión por capas |
| Letra completa | Diagnóstico → priorización → alternativas de reescritura |
| Género o estilo definido | Convenciones → brief → estructura compatible → letra |
| Beat, BPM o melodía | Brief → restricciones métricas/prosódicas → letra por secciones |

## 2. Brief conversacional

Obtén o infiere, y confirma, estos elementos:

```text
Idea central:
Ángulo propio:
Emoción específica:
Narrador y destinatario:
Tono:
Modo: narrativo / lírico / híbrido
Título provisional:
Hook:
Objeto o imagen ancla:
Arco emocional:
Contexto de escucha:
```

Si faltan tres o más elementos, no escribas todavía una canción completa. Propón un ejercicio breve:
una pregunta, tres títulos o una descripción sensorial de un objeto/escena.

## 3. Estilo y estructura

1. Si el estilo ya existe, declara sus convenciones relevantes.
2. Si no existe, ofrece hasta tres estilos con ventajas y compromisos.
3. Pide al usuario que elija uno o confirme una fusión.
4. Define secciones y función antes del borrador.

Ejemplo de mapa:

```text
Verse 1: situación e imagen concreta
Pre-Chorus: tensión o pregunta
Chorus: hook, título e idea central
Verse 2: avance o complicación
Bridge: giro de perspectiva
Final Chorus: mismo hook con nuevo peso
Outro: cierre o eco
```

## 4. Generación sin juicio

Durante esta fase, prioriza cantidad y opciones:

- Banco de imágenes sensoriales.
- Frases, títulos y variaciones de hook.
- Campos semánticos y metáforas posibles.
- Rimas potenciales sin obligarse a usarlas.

No critiques cada línea mientras se genera. Separa explícitamente la fase creativa de la fase de
edición.

## 5. Redacción por bloques

Orden recomendado:

1. Estribillo/hook si la canción es hook-driven, pop, rock, urbana o bailable.
2. Verso 1: mundo, escena o detalle.
3. Verso 2: información/emoción nueva.
4. Pre-estribillo: impulso hacia el clímax.
5. Puente: giro que resignifica.
6. Outro: cierre, repetición o retirada.

Para una canción narrativa estrófica, empieza por el arco de sucesos y escribe estrofas en orden
temporal antes de redactar el refrán.

## 6. Revisión por capas

No revises todo a la vez.

1. **Sentido:** idea, voz, destinatario, arco y concreción.
2. **Estructura:** función, contraste, hook, avance y puente.
3. **Métrica:** número de sílabas y paralelismo de líneas equivalentes.
4. **Rima:** tipo, esquema, naturalidad, importancia semántica y frescura.
5. **Prosodia/flow:** acentos, densidad, pausas y fraseo.
6. **Cantabilidad:** respiración, vocales sostenidas, articulación y trabalenguas.

Después de cada capa, ofrece cambios mínimos antes de proponer una reescritura total.

## 7. Cierre de iteración

Termina cada iteración con:

- Versión actual o fragmento corregido.
- Tres prioridades resueltas o pendientes.
- Decisiones abiertas (género, métrica, hook, tempo, punto de vista).
- Una pregunta de continuación o una prueba de lectura/canto.

## 8. Cierre: artefacto final

Cuando el usuario apruebe la dirección, entrega los tres bloques obligatorios definidos en `SKILL.md`:

1. Letra final completa.
2. Observaciones, variantes y recomendaciones.
3. Dirección musical con descripción de estilo, BPM recomendado, rango y variaciones.

No conviertas esta salida en un prompt para una IA musical. Si el usuario pide una adaptación para una
plataforma concreta en el futuro, esa será una capa adicional y separada.
